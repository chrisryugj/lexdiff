## 🧠 핵심 메모리/교훈 (machine-agnostic, 여기 embed)

### 1. Next.js 16 middleware → proxy.ts
Next.js 16부터 **`middleware.ts` 파일이 `proxy.ts`로 rename**됨.
- `export default function middleware` → `export async function proxy(request: NextRequest)`
- `export const config = { matcher: [...] }` 동일
- "middleware 없음"이라 단정 전에 `Glob "{middleware,proxy}.{ts,tsx,js}"` 양쪽 확인
- `next.config.mjs` 주석이 "middleware"라 해도 Next.js 16 프로젝트면 `proxy.ts`가 실파일

### 2. LexDiff Gemini API = Google AI Studio **유료 Tier 1+**
- Free(15 RPM / 250 RPD) 가정 금지
- Tier 1 기준: ~2,000 RPM / 10M TPM / 10K RPD
- Tier 2 기준: ~10K RPM
- 베타 현실 한계 (단일 키, 법제처/Upstash 병목 포함): **동시 30-50명 안정 / 지속 100 RPM / 일 누적 2-3K 쿼리**
- 모델 기본값: `gemini-3-flash-preview` (primary/standard), `gemini-3.1-flash-lite-preview` (router/lite) — `lib/ai-config.ts`
- `GEMINI_ROUTER_API_KEY`는 S1 Router 전용 분리 키

### 3. LexDiff Rate-Limit 모델
- **Google OAuth 로그인 필수** (미로그인 401)
- **Supabase 유저별 기능별 일 쿼터** (`lib/api-auth.ts` `requireAiAuth` → `checkAndIncrementQuota`)
- IP 기반/분당 슬라이딩 윈도우 없음 → "rate-limit 없다"고 단정 금지, 유저 쿼터가 곧 rate-limit
- BYOK(`x-user-api-key`) 형식 `/^AIzaSy[A-Za-z0-9_-]{33}$/`, 쿼터 스킵, 본인 Gemini 키 비용 자부담
- 실제 공격 비용: Google OAuth 계정 생성 필요 → bot 대량 가입 방어는 Supabase Auth 설정(CAPTCHA, 이메일 확인)에 의존
- 진단 시 순서: `lib/api-auth.ts requireAiAuth` → `lib/quota` → `supabase.user_quota` 테이블

### 4. Vercel env 함정 (이번에 발견)
- `NODE_TLS_REJECT_UNAUTHORIZED="0\n"` 잔존 중 — 코드는 CA 번들로 대체됐으나 env 청소 안 됨
- `GEMINI_API_KEY="AIzaSy...\n"` 끝 리터럴 개행 포함 — Gemini API 호출 헤더 불량 → 401 위험
- `vercel env pull` 결과를 그대로 `.env.local`로 쓰기 전에 반드시 검증:

