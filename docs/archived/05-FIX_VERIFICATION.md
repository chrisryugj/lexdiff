### 문제 2: "형법" 정확 매칭 실패

**로그 확인**:
```
🔍 [법령 검색] 검색어: "형법", 결과: N개
   결과 목록: ...
   정확 매칭: 없음 ← 이 경우!
```

**원인**: law.go.kr API 응답에 "형법"이 정확히 포함되지 않음

**해결**:
1. API 응답 확인: 실제 법령명이 "형법"인지 확인
2. 만약 "형법전" 등으로 나온다면 `search-normalizer.ts`에 별칭 추가:
   ```typescript
   {
     canonical: "형법",
     aliases: ["형법전", "형벚"],
     alternatives: [],
   }
   ```

### 문제 3: "조의" 조문이 여전히 첫 검색에서 없음

**로그 확인**:
```
📄 [파싱 완료] ... 조의 조문 X개: (비어있음)
```

**원인**: API 응답에 "조의" 조문이 없거나 파싱 실패

**진단**:
1. API 응답 확인: `/api/eflaw?lawId=...` 직접 호출
2. 조문단위 구조 확인: `조문가지번호` 필드 확인
3. 파싱 로그 확인: "📄 [파싱] 조의 조문: ..." 로그가 나타나는지

**해결**:
- 만약 API 응답에 데이터가 있는데 파싱 안되면 `parseLawJSON` 로직 수정
- 만약 API 응답에 없으면 law.go.kr API 문제

### 문제 4: Phase 7 캐시 MISS (두 번째 검색도 느림)

**로그 확인**:

❌ 캐시 MISS (검색어): "관세법 제38조"

**진단**:

   rawQuery: ❌ 없음 ← 이게 나타나면 문제!

**해결**:
- `app/page.tsx`에서 `setLawContentCache()` 호출 시 `rawQuery` 인자 전달 확인
- Line 644: `setLawContentCache(..., rawQuery)` 있는지 확인
